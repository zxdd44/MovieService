# Movie Service (Spring Boot REST API)

Данный сервис был создан и предназначен для управления информацией о фильмах.
Проект обновлен для работы с реляционной базой данных и поддерживает сложные связи между сущностями.

## Составляющие программы

1. **Spring Boot приложение**: Проект инициализирован как Spring Boot приложение с использованием Spring Web и Spring Data JPA для работы с реляционными базами данныъ.
2. **Расширенная область (Domain)**: реализовано 5 связанных сущностей — `Movie, Director, Genre, Studio, Review`, настроены связи OneToMany и ManyToOne.
3. **Реализация REST API**:
    * Реализованы все CRUD операции (Create, Read, Update, Delete) для управления каталогом.
    * Поддерживается получение данных через '@PathVariable' и гибкая фильтрация через '@RequestParam'.
4. **Слои приложения**:
    * **Controller**: `MovieController` — обрабатывает внешние HTTP-запросы.
    * **Service**: `MovieService` — содержит бизнес-логику поиска и фильтрации.
    * **Repository**: `MovieRepository` — Доступ к БД через JPA.
5. **DTO и Mapping**:
    * Объект `MovieDto` используется для возврата данных пользователю, скрывая детали реализации (например, мапит статус в числовой код).
    * `MovieMapper` инкапсулирует логику преобразования сущности `Movie` в `MovieDto`.

## Эндпоинты (CRUD)

### 1. Базовые операции
* 'POST /api/movies' — **создание нового фильма**.
* 'GET /api/movies/{id}' — **получение по ID**.
* 'PUT /api/movies/{id}' — **обновление данных**.
* 'DELETE /api/movies/{id}' — **удаление**.

### 2. Фильтрация
* **URL**: `GET /api/movies/filter?director=...`
* **Пример запроса**: `GET http://localhost:8080/api/movies/filter?director=Christopher%20Nolan`

## Структура данных (Status Codes)
В API статусы фильмов передаются в виде целых чисел:
* `0` — NOT_WATCHED (Не просмотрено)
* `1` — WATCHED (Просмотрено)
* `2` — DEFERRED (Отложено)
* `3` — ABANDONED (Брошено)

---

## Модель данных (Entity Relationship)
В проекте реализовано 5 основных сущностей:
* **Movie**: Основная сущность
* **Director**: Режиссер (Связь OneToMany с Movie).
* **Genre**: Жанр (Связь ManyToMany с Movie).
* **Studio**: Пользователь сервиса.
* **Review**: Отзыв пользователя (Связь ManyToOne к Movie).

## Управление связями (Fetch & Cascade)
* 'FetchType.LAZY': Используется для всех коллекций (например, в Director.movies), чтобы избежать избыточной нагрузки на БД.
* 'CascadeType.ALL': Настроен для связи Movie -> Review. При удалении фильма автоматически удаляются все связанные с ним отзывы.

## Решение проблемы N+1
Для оптимизации производительности при получении списка фильмов с их авторами реализовано решение
@EntityGraph: Декларативное описание графа зависимостей в репозитории.

## Транзакционность и целостность
* **Без @Transactional**: При ошибке во время сохранения отзыва сам фильм может остаться в базе (частичное сохранение), что нарушает логику.
* **С @Transactional**: Любая ошибка вызывает полный Rollback, гарантируя, что база данных останется в консистентном состоянии.
